BOMI PAY DATABASE SCHEMA PACK

========================================
1. OVERVIEW
========================================
This schema defines the core data structure for Bomi Pay,
an AI-driven payment intelligence and orchestration platform.

Database Type: PostgreSQL (Primary)
Supporting: Redis (cache), Object Storage (files), Warehouse (analytics)

========================================
10. INDEXING STRATEGY
========================================

Recommended indexes:
- transactions(merchant_id, status)
- transactions(provider_transaction_id)
- transactions(external_reference)
- reconciliation_results(run_id)
- disputes(transaction_id)
- alerts(merchant_id, status)

========================================
11. NOTES
========================================

- All financial records should be append-only where possible
- Use UUIDs for IDs in production
- Store raw provider payloads in object storage
- Use JSON fields for flexible metadata
- Ensure idempotency in transaction ingestion
