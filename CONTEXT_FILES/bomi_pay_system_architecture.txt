BOMI PAY SYSTEM ARCHITECTURE

1. CORE POSITION
Bomi Pay is an AI-driven payment intelligence and orchestration platform for Nigeria.
It starts as a control layer above payment providers and expands into routing, disputes,
fraud, collections, payouts, and gateway capabilities.

2. ARCHITECTURE PRINCIPLE
Do not begin as a bank replacement.
Begin as an intelligence and control layer above:
- payment gateways
- banks
- USSD channels
- biller platforms
- payroll systems
- merchant collection systems

11. CORE SERVICES IN DETAIL

11.1 Identity and Access Service
- merchant sign-up
- user roles
- team access
- API keys
- OAuth/token management
- session management

Roles:
- merchant owner
- finance staff
- operations staff
- support agent
- developer
- super admin

11.2 Merchant Service
- merchant profile
- business type
- settlement preferences
- linked providers
- bank accounts
- billing plan
- KYC/KYB status

11.3 Transaction Ingestion Service
Inputs:
- provider webhooks
- provider polling APIs
- CSV uploads
- bank statements
- internal transaction creation

Responsibilities:
- validate payload
- assign idempotency key
- normalize event
- publish event
- store raw and normalized records

11.4 Payment Event Normalization Service
Canonical fields:
- transaction_id
- external_reference
- merchant_id
- customer_id
- source_provider
- payment_channel
- amount
- currency
- fees
- status
- failure_reason
- initiated_at
- updated_at
- settled_at
- payer details
- receiver details
- metadata

11.5 Reconciliation Service
Responsibilities:
- match payment to invoice/order/payroll/bill
- detect underpayment
- detect overpayment
- detect duplicates
- detect hanging payments
- detect settlement mismatch
- generate reports

Matching methods:
- exact reference match
- amount and customer match
- date window match
- fuzzy match with confidence score

11.6 Dispute and Resolution Service
Responsibilities:
- open issues automatically
- classify dispute type
- track status
- escalate cases
- notify merchant/customer
- attach evidence

Dispute types:
- debited not credited
- duplicate debit
- settlement mismatch
- chargeback
- fraud suspicion
- reversal delay

11.7 Ledger Service
Responsibilities:
- record platform-side financial events
- track fees
- track commissions
- track payout states
- create auditable trail

Recommended model:
- append-only double-entry ledger

11.8 Provider Orchestration Service
Responsibilities:
- choose provider by rules or AI
- retry with fallback provider
- detect provider downtime
- score provider reliability
- route by payment type

11.9 Notification Service
Channels:
- email
- SMS
- WhatsApp
- push notification
- webhook callbacks

11.10 Risk and Fraud Service
Inputs:
- transaction velocity
- device fingerprint
- unusual amount patterns
- provider behavior
- beneficiary changes
- duplicate attempts
- geo anomalies

Outputs:
- risk score
- flag reason
- recommended action
- explainable summary

12. AI ARCHITECTURE

12.1 AI Support Assistant
Answers:
- why did this payment fail
- has this payment been settled
- what is the likely cause
- what should the merchant do next
- what should be escalated

12.2 AI Reconciliation Assistant
Examples:
- explains unmatched payments
- explains short settlements
- summarizes likely outage patterns

12.3 AI Anomaly Detector
Examples:
- spike in failures
- repeated timeout from one provider
- duplicate callback patterns
- unusual payroll failure cluster

12.4 AI Routing Optimizer
Inputs:
- historical success rate
- latency
- provider fees
- outage pattern
- amount band
- transaction type

Output:
- recommended provider route

12.5 AI Merchant Insights Engine
Outputs:
- top failure windows
- lost revenue from failures
- customer drop-off by method
- best-performing collection channel
- settlement delay trend

13. DATA ARCHITECTURE

18. SECURITY ARCHITECTURE

- encryption in transit
- encryption at rest
- HMAC webhook verification
- strict audit logging
- RBAC
- least privilege
- provider credential vaulting
- idempotency protection
- tamper-evident records

Operational protections:
- suspicious login alerts
- API abuse detection
- replay attack protection
- payout approval controls
- dual approval for high-value actions
